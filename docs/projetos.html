ConsultaVid- 2023-1

Sobre

Parceiro acadêmico: Faculdade de Tecnologia de São José dos Campos - Prof. Jessen Vidal.
Link para o repositório do projeto: github.com/APIEquipe/EquipeTechEmpireAPI


Neste projeto nosso parceiro acadêmico propôs um projeto com ênfase no ocorrido nos anos anteriores, com a proposta de um sistema capaz de trazer dados sobre possíveis casos de covid longa. A COVID longa são sintomas encontrados em pacientes que tiveram COVID-19 e que mesmo após terem se curado da doença, apresentam sintomas prolongados como falta de ar que em alguns casos é necessário o uso de bombinha de ar para o tratamento. O sistema abordou um processo de coleta de dados sobre os sintomas prolongados do covid-19, com foco na região do vale do paraíba foi desenvolvido funcionalidades para analisar os dados coletados, podendo assim exibir para o usuário os sintomas do covid longa e os momentos de picos durante a pandemia.

Fonte: github.com/APIEquipe/EquipeTechEmpireAPI/blob/main/relatorios-sprint/sprint4.md

Sobre

Parceiro acadêmico: Faculdade de Tecnologia de São José dos Campos - Prof. Jessen Vidal.
Link para o repositório do projeto: github.com/BananaaScript/BetterCallUs

O sistema também contava com uma página principal com problemas frequentes, auxiliando o usuário a encontrar a solução para o seu problema sem ter que criar um ticket para o suporte, o suporte que tinha todo o controle dos tickets, o usuário podia enviar um ticket descrevendo o tipo de problema que ele estava enfrentando e qual dispositivo sofreu o problema, sendo assim o suporte respondia esse ticket enviado e o usuário podia conferir a resposta enviada.

Fonte: github.com/BananaaScript/BetterCallUs/tree/main

SGA - 2023-2

Sobre

Parceiro acadêmico: Youtan
Link para o repositório do projeto: github.com/BananaaScript/SGA


Nesse projeto, nosso parceiro acadêmico propôs um sistema de gerenciamento de ativos para empresas, o sistema de consiste em gerenciar ativos de uma empresa, o sistema conta com relatórios dos ativos e divisões por modelo e categoria, usuários administradores são capazes de gerenciar o sistema como um todo, permitindo eles de cadastrar modelos, categorias, ativos e usuários. O sistema possuí a funcionalidade de notificações dentro do sistema e por email, sendo assim o usuário que possuí um ativo vai receber uma notificação no sistema e no seu email quando o ativo em sua posse estiver próximo da sua data de manutenção preventiva.


Resumindo, o sistema é destinado para empresas e seus funcionarios, onde a empresa cadastra seus ativos, define suas propriedades e define um funcionário proprietário, os funcionários dessa empresa são capazes de visualizar os ativos em sua posse e visualizar a data de manutenção preventiva do ativo em sua posse, quando o ativo entrar em manutenção a posse retorna para a empresa.

Fonte: github.com/BananaaScript/SGA

Meteorological Data Collector - 2023-2

Sobre

Parceiro acadêmico: Tecsus
Link para o repositório do projeto: github.com/BananaScripts/Meteorological-Data-Collector

A proposta seria de um desenvolvimento de uma aplicação web que auxilie na coleta, processamento e visualização de dados coletados por estações meteorológicas, no qual o sistema disponibiliza funcionalidades para visusalizar os dados das estações, sendo possivel aplicar filtros se desejado, sistema de alerta para momentos em que um parametro seja excedido, uma página de educação para incentivar alunos do ensino médio a aprender.
